Verify credentials presented by mobile wallets

It is a bridge between a standard OIDC client application and a digital identity wallet.

Integration can be achieved with any framework (NodeJS, Python, PHP, C, C+, GO, Ruby,..Webflow, Wordpress,..;) with the help of a standard OIDC client lib.

All parameters are managed through the console of the platform, so developer only need to setup the issuer, client_id and client_secret parameters in their code.

Implementation time for POC : a couple of hours.

Issue credentials to mobile wallets

Issuance of credentials is done through a unique call to an API and a redirect of the user browser.

Through the API developers will provide the main VC data and some options.

Other parameters are setup through the console of the platform.

A swagger UI is available with example.

Implementation time for POC : 1 to 2 days.
